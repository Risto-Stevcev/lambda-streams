Module Lambda_streams__Sync

Generalizes the notion of a syncronous readable stream. It has some interesting properties compared to traditional (potentially asyncronous) streams:

  • Since the function returns a Signal.t, it implies that the function is both side-effecting and utilizes a closure because it can return a different result on each invocation.

  • Like traditional streams:

    • It is a transducer

  • Unlike traditional streams:

    • There is no inversion of control: the stream is not passed another stream to which it communicates with.
    • There is direct control: a value is available at all times because the stream can be invoked to recieve a new value. It's pull-based.
    • As a result, functions like filter can't be implemented, but until can.
